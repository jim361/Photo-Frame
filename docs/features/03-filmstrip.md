# 03. 필름 스트립 (썸네일 선택/정렬/삭제)

## 목적

롤 단위 작업의 중심 UI. 불러온 컷들을 필름 스트립처럼 나란히 놓고, 클릭으로 오가며 확인하고, 드래그로 순서를 바꾼다. 스트립 순서가 곧 내보내기 순서다.

## 현재 동작

- 하단 고정 바(`#strip`)에 높이 52px, 사진 비율에 따라 폭 36~88px인 칩. 빈 상태면 안내 문구를 CSS `::before`로 표시.
- 칩에는 원본 전체가 잘리지 않게 `object-fit: contain`으로 표시한 objectURL 썸네일 + 호버 시 × 삭제 버튼. (프레임 번호 표기는 롤 번호 기능 제거와 함께 삭제됨)
- **선택**: 클릭 → `state.current` 변경, 선택 칩만 불투명+주황 테두리, `refresh(true)`.
- **삭제**: × 클릭 → objectURL revoke + ImageBitmap close, 배열에서 제거, current 보정(마지막 삭제 시 앞으로 이동).
- **정렬**: HTML5 Drag and Drop.
  - `dragover`에서 칩의 좌/우 절반 판정 → 주황 세로선 인디케이터를 앞/뒤에 표시
  - `drop`에서 `from < to`일 때 인덱스 −1 보정(뽑힌 자리), 이동 후 **선택 중이던 사진을 계속 선택 유지** (객체 참조로 재탐색)

## 설계 결정

- **HTML5 DnD 채택** — 라이브러리 없이 데스크톱에서 충분. 터치 DnD가 안 되는 것은 알려진 트레이드오프.
- **전체 재렌더(`innerHTML = ''`) 방식** — 36장 수준에서는 성능 문제가 없어 diff 없이 단순하게 유지.
- 썸네일을 원본 objectURL로 표시 — 구현 단순성 우선. 메모리 개선 과제로 남김.

## 알려진 한계

- **터치 기기에서 드래그 정렬 불가** (HTML5 DnD 한계). pointer 이벤트 재구현 필요.
- 썸네일이 원본 이미지를 그대로 참조 — 고해상도 36장 로드 시 메모리 부담 (ARCHITECTURE.md 메모리 항목).
- 다중 선택 없음 — 장별 오버라이드([07](07-per-photo-override.md)) 설계와 함께 재검토.
