# 02. 줌/팬 뷰포트

## 목적

합성 결과를 라이트테이블 위에서 확인한다. 전체 구도(화면 맞춤)와 디테일(1:1로 엣지 프린트 확인)을 오가며, 촬영 정보를 타이핑하는 동안에도 보고 있던 위치를 잃지 않아야 한다.

## 현재 동작 (v3)

CSS `transform: translate(x,y) scale(s)` 기반. 상태는 `zoom { scale, fit, x, y, mode }`.

- **모드 2개**: `fit`(화면 맞춤 유지) / `free`(사용자가 줌·팬한 상태). 뷰포트 리사이즈 시 `fit` 모드만 ResizeObserver로 재맞춤.
- **화면 맞춤 배율**: `min((뷰포트폭−48)/캔버스폭, (뷰포트높이−72)/캔버스높이, 1)` — 여백 확보, 원본보다 확대하지 않음.
- **휠 줌**: 커서 위치 고정점 유지, 배율 1.12/스텝. 범위 0.03×~4×.
- **드래그 팬**: pointer 이벤트 + `setPointerCapture`. 줌바·드롭존 위에서는 시작 안 함.
- **툴바**: − / 배율%(클릭=맞춤) / ＋ / 맞춤 / 1:1. 버튼 줌은 1.25/스텝, 뷰포트 중앙 고정점.
- **키보드**: `+ − 0 1` (input/select/textarea 포커스 중에는 무시).
- **줌 유지 규칙**: 정보 입력 → `refresh(true)`로 free 줌 유지. 스타일 변경 → 비율이 바뀌므로 맞춤 리셋. 사진 선택 변경 → `refresh(true)` (같은 롤이면 비율이 대체로 같으므로 유지).

## 설계 결정

- **canvas 재렌더 + transform 분리** — 줌·팬은 transform만 바꾸므로 픽셀 재합성 없이 즉각적. 재합성은 정보가 바뀔 때만.
- **커서 고정점 수식**: 고정점이 가리키는 캔버스 좌표 `(px−x)/s`를 새 배율에서도 같은 화면 위치에 유지. 표준 기법이지만 fit/free 모드 구분과 결합한 것이 이 도구의 요점 — "사용자가 만진 줌은 존중, 아니면 항상 전체 보기".
- 최소 배율 0.03×: 매우 큰 스캔(1만px대)도 화면 맞춤이 가능해야 해서.

## 알려진 한계

- **핀치 줌 없음** — 모바일에서 휠·키보드가 없어 툴바 버튼만 가능. pointer 이벤트 기반이라 two-pointer 거리 계산으로 확장 가능.
- transform scale 확대 시 캔버스가 블러 처리됨(브라우저 보간) — 1:1 이상 확대는 픽셀 확인용으로는 부정확.
