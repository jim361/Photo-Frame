# 02. 줌/팬 뷰포트

## 목적

합성 결과를 라이트테이블 위에서 확인한다. 전체 구도(화면 맞춤)와 디테일(1:1로 엣지 프린트 확인)을 오가며, 촬영 정보를 타이핑하는 동안에도 보고 있던 위치를 잃지 않아야 한다.

## 현재 동작 (v3)

CSS `transform: translate(x,y) scale(s)` 기반. 상태는 `zoom { scale, fit, x, y, mode }`.

- **모드 2개**: `fit`(화면 맞춤 유지) / `free`(사용자가 줌·팬한 상태). 뷰포트 리사이즈 시 `fit` 모드만 ResizeObserver로 재맞춤.
- **화면 맞춤 배율**: `min((뷰포트폭−48)/캔버스폭, (뷰포트높이−72)/캔버스높이, 1)` — 여백 확보, 원본보다 확대하지 않음.
- **휠 줌**: 커서 위치 고정점 유지, 배율 1.12/스텝. 범위 0.03×~4×.
- **드래그 팬**: pointer 이벤트 + `setPointerCapture`. 줌바·드롭존 위에서는 시작 안 함.
- **툴바**: − / 배율%(클릭=맞춤) / ＋ / 맞춤 / 1:1 / **경계** / **↶ 돌아가기**. 버튼 줌은 1.25/스텝, 뷰포트 중앙 고정점.
- **경계 토글 + 드래그 배치**: 미리보기 위에 카메라/렌즈/필름·기타/설정/날짜/메모/로고·서명의 실제 렌더 영역을 주황 점선 박스 + 라벨로 표시하고, **박스를 잡아 끌면 위치가 바뀐다**. 드래그 중 다른 요소의 경계·중앙이나 캔버스 중앙과 수평/수직이 맞으면 파란 가이드라인이 뜨며 스냅. 더블클릭으로 해당 요소 위치 초기화. 오프셋은 사진 폭 W 비율로 스타일·방향별 프로파일(각 요소 `DX/DY`)에 저장 — 해상도가 달라도 같은 자리이며 다른 프레임 스타일이나 사진 방향에 섞이지 않는다. 경계가 켜져 있으면 조정 부위 드래그는 배치·크기, 빈 곳 드래그는 팬. 가로 인스탁스는 스택·회전 글씨 모두 개별 드래그를 지원한다. 세로 필름 스트립의 회전 엣지 프린트는 하나의 자동 배치 영역으로 표시되고 개별 드래그는 지원하지 않는다. render()가 `info.collect`일 때 요소 박스를 수집(`info.boxes`)하고 미리보기에만 오버레이 — 내보내기에는 포함되지 않는다. **기본 켜짐** — 줌바 '경계'로 끌 수 있고 선택은 `frame.ui`에 저장.
- **프레임 크기 핸들**: 하단 공간은 텍스트 영역 바깥 모서리의 선(스타일에 따라 아래·위·우측), 테두리 여백(인스탁스)은 **사진 우상단 모서리의 사각 그립**. 사진 크기를 조절하는 감각에 맞춰 그립을 왼쪽·아래(SW)로 끌면 여백이 커지고 오른쪽·위(NE)로 끌면 작아진다. 드래그 중 파란 강조 + 현재 % 라벨, 1% 단위, fit 모드에서는 1400px 축소본으로 렌더해 버벅임 방지.
- **1단계 레이아웃 돌아가기**: 마지막 요소 이동, 프레임 크기 변경, 레이아웃 수치 변경, 프리셋 적용, 더블클릭·버튼 초기화 직전의 현재 스타일 세로·가로 프로파일 한 쌍을 세션 메모리에 보관한다. 같은 스타일에서 `↶ 돌아가기`를 누르면 복원하고 슬롯을 비운다. 다른 스타일에서는 버튼이 비활성화되며 원래 스타일로 돌아오면 다시 사용할 수 있다. 프로젝트를 열거나 새로고침하면 이 임시 기록은 지운다.
- **호버 피드백**: 조정 가능한 부위 위에 마우스를 올리면 해당 핸들·박스가 파랗게 강조되고 커서가 바뀐다(박스 `move`, 모서리 그립 `nesw-resize`, 선 `ns/ew-resize`). 히트 우선순위는 호버와 클릭이 동일 — 모서리 그립 → 경계선 → 요소 박스 → 팬.
- **키보드**: `+ − 0 1` (input/select/textarea 포커스 중에는 무시).
- **줌 유지 규칙**: 정보 입력 → `refresh(true)`로 free 줌 유지. 스타일 변경 → 비율이 바뀌므로 맞춤 리셋. 사진 선택 변경 → `refresh(true)` (같은 롤이면 비율이 대체로 같으므로 유지).

## 설계 결정

- **canvas 재렌더 + transform 분리** — 줌·팬은 transform만 바꾸므로 픽셀 재합성 없이 즉각적. 재합성은 정보가 바뀔 때만.
- **커서 고정점 수식**: 고정점이 가리키는 캔버스 좌표 `(px−x)/s`를 새 배율에서도 같은 화면 위치에 유지. 표준 기법이지만 fit/free 모드 구분과 결합한 것이 이 도구의 요점 — "사용자가 만진 줌은 존중, 아니면 항상 전체 보기".
- 최소 배율 0.03×: 매우 큰 스캔(1만px대)도 화면 맞춤이 가능해야 해서.

## 알려진 한계

- **핀치 줌 없음** — 모바일에서 휠·키보드가 없어 툴바 버튼만 가능. pointer 이벤트 기반이라 two-pointer 거리 계산으로 확장 가능.
- transform scale 확대 시 캔버스가 블러 처리됨(브라우저 보간) — 1:1 이상 확대는 픽셀 확인용으로는 부정확.
