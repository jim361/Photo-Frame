# 05. 프리셋

## 목적

자주 쓰는 장비·필름 조합을 한 번 저장해 두고 칩 클릭 한 번으로 적용한다. 롤이 바뀔 때마다 "F5 + 50mm + Portra"를 다시 타이핑하지 않는 것이 목적. 필름 사진가는 조합이 몇 개로 수렴하는 경향이 있어 칩 몇 개면 충분하다.

## 현재 동작 (v3)

- **저장**: "현재 설정 저장" → `prompt()`로 이름 입력(기본값: 채워진 항목을 `·`로 연결) → `{ name, body, lens, film, style, show, lines }` 저장 (표시 설정 포함).
- **적용**: 칩 클릭 → 바디/렌즈/필름 입력값 + 프레임 스타일 세그먼트 갱신, 미리보기 갱신, 토스트.
- **삭제**: 칩 안의 × 클릭.
- **저장소**: `store` 래퍼로 localStorage `frame.presets`. 접근 실패(iframe 미리보기 등) 시 메모리 폴백 — 저장 UI는 동작하되 세션 한정.
- **시드**: 최초 실행 시 제작자 기준 프리셋 3개(F5·50mm·Portra 등)가 채워져 있다.
- 프리셋의 장비·필름 값은 datalist 자동완성 후보로도 흡수된다.

## 설계 결정

- **prompt() 사용** — v3 프로토타입에서 이름 입력 모달을 만드는 비용을 아꼈다. 마이그레이션 시 인라인 입력으로 교체 예정.
- **톤(tone)·롤 번호는 저장하지 않음** — 프리셋은 "장비 조합"이지 "롤 상태"가 아니다. 롤 번호는 매 롤 바뀌므로 제외.
- 시드 프리셋을 코드에 두는 이유: 빈 화면보다 예시가 있는 화면이 사용법을 설명한다.

## 알려진 한계

- 표시 설정이 없는 구버전/시드 프리셋 적용 시 현재 표시 설정을 유지한다 (의도된 하위 호환).
- 기기 간 이동 불가 — JSON 내보내기/가져오기 필요.
- `prompt()`는 일부 임베드 환경에서 차단된다.
