# 05. 프리셋 (장비·레이아웃)

## 목적

자주 쓰는 카메라/렌즈/필름 값과 프레임별 배치를 저장해 두고 클릭 한 번으로 적용한다. 필름 사진가의 장비는 몇 개로 수렴하므로, 장비는 조합 단위가 아니라 **항목 단위 팔레트**로 관리하고 레이아웃은 스타일별 세로·가로 한 쌍으로 관리한다.

## 현재 동작

- **구조**: 카메라 / 렌즈 / 필름 3열 그리드. 각 열은 저장된 값 칩 목록 + `＋` 버튼.
- **저장**: `＋` 클릭 → 해당 입력 필드의 현재 값을 그 열에 저장. 빈 값·중복은 토스트로 거절.
- **적용**: 칩 클릭 → 해당 입력 필드에만 값 반영 (`input` 이벤트 발생 → 자동 저장 + 미리보기 갱신). 다른 항목은 건드리지 않는다.
- **삭제**: 칩 안의 × 클릭.
- **저장소**: `store` 래퍼로 localStorage `frame.gear` — `{ body: [], lens: [], film: [] }`. 접근 실패 시 메모리 폴백. 구버전 조합 프리셋(`frame.presets`)이 남아 있으면 첫 실행 때 값만 이전한다.
- **시드**: 배포본은 빈 팔레트로 시작한다. 제작자 장비가 다른 사용자 설정에 섞이지 않도록 제거했으며, 기존 브라우저의 저장값은 그대로 유지한다.
- 팔레트 값은 datalist 자동완성 후보로도 쓰인다.
- 긴 값은 칩에서 말줄임(ellipsis) 처리, 전체 값은 툴팁으로 표시.
- **레이아웃 프리셋**: 현재 프레임 스타일의 세로·가로 프로파일 한 쌍을 이름 붙여 `frame.layouts`에 저장한다. 프리셋 목록은 현재 스타일에 속한 항목만 보여 주며, 적용해도 다른 프레임 스타일의 배치는 바꾸지 않는다. 적용 직전 프로파일은 줌바의 1단계 `돌아가기`로 복원할 수 있다. 스타일 필드가 없는 구버전 프리셋은 현재 스타일에서 계속 적용할 수 있다.

## 설계 결정

- **조합 프리셋 → 항목별 팔레트 전환** (2026-07, 사용자 결정) — 조합 저장은 "이름 짓기" 비용이 들고, 조합 수는 항목 수의 곱으로 늘어난다. 항목별 저장이면 이름도 필요 없고 `prompt()`도 사라진다.
- 스타일·표시 설정은 프리셋 범위에서 제외 — 설정 자동 저장(`frame.settings`)이 이미 세션 간 유지를 담당한다.

## 알려진 한계

- 기기 간 이동 불가 — JSON 내보내기/가져오기 필요.
- 3열 고정이라 좁은 화면에서 칩이 더 짧게 잘린다 (툴팁으로 보완).
