# 01. 프레임 렌더링 엔진

## 목적

필름 스캔본 위에 프레임과 촬영 정보를 합성해 "인스타에 바로 올릴 수 있는 한 장"을 만든다. 사용자는 스타일 하나를 고르고 정보를 채우면 되고, 레이아웃·타이포·비율은 도구가 책임진다.

## 현재 동작 (v3)

`render(img, o): canvas` — 원본 크기 기준으로 새 캔버스에 합성한다. 미리보기와 내보내기가 같은 함수를 공유한다.

**공통 텍스트 규칙**
- `meta(o)`: 바디·렌즈를 `  ·  `로 연결. 빈 항목은 자동 생략 (`filter(Boolean)`)
- `frameLabel(o)`: 시작 프레임 + 사진 순번, 2자리 패딩 + `A` (예 `07A`) — 필름 엣지 표기 관례
- `trackedText`: `ctx.letterSpacing` 지원 시 사용, 미지원 브라우저는 글자 단위 수동 렌더 폴백

**스타일 4종** (수치 근거는 [DESIGN.md](../DESIGN.md), 두 줄 레이아웃은 [04. 촬영 정보](04-shot-info.md))

| 스타일 | 캔버스 크기 (한 줄 / 두 줄) | 내용 |
|---|---|---|
| `film` | W × (H + 0.242W / 0.290W) | 위아래 검정 바 + 퍼포레이션 2줄(아래 줄은 사진 바로 아래 고정), 아래 바에 프레임 번호(주황, 좌) + 장비·필름(회색, 우, 대문자). 두 줄이면 아래 바 확장(1.75bar→2.3bar) 후 장비/필름 분리 |
| `instax` | 1.11W × (H + 0.055W + 0.27W) | 아이보리 카드, 사진 둘레 옅은 스트로크, 하단 여백에 장비(중앙)/필름(중앙)/프레임 번호(우하단). 카드 디자인상 항상 2단 — 레이아웃 옵션 무시 |
| `bottom` | W × (H + 0.078W / 0.112W) | 사진 아래 여백 띠 — 장비·필름(좌) + 프레임 번호(주황, 우). 밝은/어두운 톤. 글자 크기는 줄 수와 무관하게 동일(0.0234W) |
| `top` | 위와 동일 | `bottom`과 동일하되 띠가 위 |

**안전장치**
- `safeScale(W,H,o)`: 스타일별 합성 결과 크기를 사전 계산해 브라우저 캔버스 한계(iOS 4096² 면적 / 그 외 변 16384·면적 1.2억px) 초과분만큼 소스를 축소. 축소 시 미리보기 토스트(장당 1회)와 내보내기 완료 메시지로 안내.
- EXIF 회전: `createImageBitmap(file, { imageOrientation: 'from-image' })`로 디코드 시 확정, 미지원 브라우저는 `<img>` 폴백.

`downscale(cv, longEdge)`: 내보내기 시 긴 변 기준 축소 (2048/1440/원본), `imageSmoothingQuality: 'high'`.

## 설계 결정

- **원본 해상도로 합성 후 마지막에 축소** — 텍스트·퍼포레이션이 축소 과정에서 자연스럽게 안티앨리어싱된다. 처음부터 작게 그리면 얇은 스트로크가 뭉개진다.
- **모든 수치를 사진 폭 W의 비율로** — 어떤 해상도의 스캔이 와도 프레임 비율이 동일하다.
- **DOM 캔버스 사용 (OffscreenCanvas 아님)** — v3 시점에는 워커 분리 이득보다 단순함을 택했다. 2단계에서 순수 함수로 분리할 때 재검토.
- SVG/CSS 합성이 아닌 Canvas인 이유: 결과물이 픽셀 이미지 파일이어야 하고, 미리보기=결과물 일치가 필요하다.

## 알려진 한계

- **ICC 프로파일 무시** — Adobe RGB 등 임베디드 프로파일이 있는 스캔은 색이 틀어질 수 있다. sRGB 변환 후 사용 권장 (ARCHITECTURE.md 참고).
- **한 줄 모드에서 긴 장비명 겹침** — 좌측 프레임 번호와 우측 장비·필름 텍스트가 길면 겹친다. 두 줄 레이아웃이 해법이며 UI에서 선택 가능. 자동 축소/자동 두 줄 전환은 미구현.
- 캔버스 한계 검출이 UA 기반 휴리스틱 — 실측(프로브) 방식이 아니라 보수적 상수를 쓴다.
