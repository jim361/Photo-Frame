# 08. 다국어 (ko/en) [미구현]

## 목적

UI 언어를 한국어/영어로 전환한다. GitHub 공개 프로젝트로서 영어 사용자가 데모를 열었을 때 쓸 수 있어야 한다. 결과물(캔버스 텍스트)은 장비명·필름명 그대로라 언어 중립이므로, 대상은 **UI 문자열만**이다.

## 요구사항

- 대상: 라벨, 버튼, 안내 문구, 토스트, 툴팁, `placeholder`, 빈 상태 문구(CSS `::before` 포함 — JS로 옮기거나 `data-` 속성화 필요).
- 언어 전환 UI(헤더 또는 설정), 선택은 localStorage 저장, 초기값은 `navigator.language` 기반 (ko* → ko, 그 외 en).
- 외부 i18n 라이브러리 없이 사전 객체 방식 (의존성 최소화 정책).
- `<html lang>` 동기화.
- 시드 프리셋 이름 등 데이터성 문자열은 번역하지 않는다.

## 설계 메모

- 2단계(TS 마이그레이션) 이후 구현하는 것이 효율적 — 문자열이 컴포넌트로 흩어지기 전에 키 체계를 잡으면 재작업이 준다.
- 날짜/장소 필드(3단계)가 추가되면 날짜 포맷 로캘 처리도 이 기능의 범위에 들어온다.

## 개선 계획

- [ ] 3단계: 문자열 사전 추출 → 전환 UI → 자동 감지

## 수용 기준

- [ ] 모든 UI 문자열이 ko/en 전환된다 (하드코딩 잔존 0)
- [ ] 선택 언어가 새로고침 후 유지된다
- [ ] 브라우저 언어가 영어면 첫 방문 시 영어로 뜬다
- [ ] 결과물 이미지는 언어 설정과 무관하게 동일하다
