# 06. 내보내기 + 파일 시스템

## 목적

롤 전체를 한 번에, 지정한 폴더로, 업로드하기 좋은 크기로 저장한다. "36장을 다운로드 폴더에서 주워 담는" 경험을 피하는 것이 목적.

## 현재 동작 (v3)

- **비율 맞춤**: 인스타 자동(기본) / 원본 비율 / 4:5 / 1:1 / 1.91:1 + 여백 색(흰색/아이보리/검정). 인스타 자동은 피드 허용 범위(4:5~1.91:1)를 벗어날 때만 `padToRatio()`로 중앙 배치 패딩 — 업로드 강제 크롭 방지. 미리보기에도 동일 적용(보이는 대로 저장), 캔버스 한계 계산(`safeScale`)에도 반영.
- **크기**: 긴 변 2048px(인스타 권장) / 1440px / 원본. 패딩 후 `downscale()`에서 초과분만 축소.
- **포맷**: PNG(기본, 무손실) / JPEG + 품질 50~100(기본 92). JPEG 선택 시에만 품질 필드 표시. 확장자(.png/.jpg)가 형식을 따라간다.
- **저장 위치**:
  - `showDirectoryPicker({ mode:'readwrite' })` 지원 시(Chromium) "저장 폴더 선택" → 이후 모든 내보내기가 그 폴더로 직행.
  - 미지원(Firefox/Safari/iframe) 시 버튼 비활성 + 경고 표시, `<a download>` 다운로드 폴백. 장당 350ms 딜레이로 브라우저 다운로드 연타 차단 회피.
- **파일명**: `원본이름_framed.{확장자}` (예 `scan01_framed.jpg`). 롤·프레임 번호 기반 명명은 해당 필드 제거와 함께 삭제됨.
- **범위**: 전체 내보내기 / 현재 장만 내보내기.

## 설계 결정

- **File System Access API 우선** — 롤 단위 흐름의 완성. 폴더 핸들은 세션 동안만 유지(권한 모델상 재선택 필요).
- **원본 이름 유지 + `_framed` 접미사** — 스캔 파일명과의 대응이 바로 보인다. 원본 정렬 순서가 곧 결과물 정렬 순서.
- **PNG 기본** — 텍스트·퍼포레이션의 선명도 보존. 인스타는 어차피 재압축하므로 대용량 롤은 JPEG 옵션이 실용적.

## 알려진 한계

- 다운로드 폴백에서 여러 장 저장 시 브라우저가 "여러 파일 다운로드" 허용을 물을 수 있다.
- 같은 파일명 존재 시 폴더 저장은 덮어쓴다 (다운로드 폴백은 브라우저가 `(1)` 붙임).
- 진행 표시 없음 — 36장 내보내기 중 UI가 조용하다.
