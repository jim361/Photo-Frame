# 04. 촬영 정보 + 표시 설정

## 목적

바디/렌즈/필름/롤 번호/시작 프레임을 한 번 입력하면 롤 전체에 적용된다. 핵심은 **불확실한 정보를 표시하지 않을 수 있는 것** — 필름 사진은 현상까지 시차가 있어 렌즈 기억이 흐려지는 일이 잦고, 틀린 정보를 박제하느니 빼는 쪽을 지원한다 (CONCEPT.md 원칙 2).

## 현재 동작 (v3)

- **입력 필드**: 바디 / 렌즈 / 필름 (텍스트 + `datalist` 자동완성), 롤 번호 / 시작 프레임 (숫자).
- **자동완성 소스**: SEED 기본값(Nikon F5, 렌즈 2종, 필름 5종) + 저장된 프리셋에서 추출한 값의 합집합.
- **빈 칸 = 생략**: 렌더 시 `filter(Boolean)`으로 빈 항목이 텍스트에서 빠진다. 현재는 이것이 유일한 "표시 안 함" 수단.
- **프레임 번호**: `시작 프레임 + 사진 순번` 자동 증가. 시작 프레임을 비우면 번호 자체를 표시하지 않음.
- **즉시 반영**: 모든 입력에 `input` 이벤트 → 스트립(번호)과 미리보기 갱신, 줌 유지.
- 초기값은 하드코딩(F5 / 50mm f/1.8G / Portra 400). **localStorage 저장 없음** — 새로고침하면 초기값으로 돌아간다.

## 설계 결정

- **자유 텍스트 + datalist** (select 아님) — 장비는 무한히 다양하고, 자동완성은 반복 입력만 줄이면 된다. 프리셋에서 자동완성을 채우는 것으로 "쓸수록 편해지는" 구조.
- **롤 전역 입력이 기본** — 필름은 롤 단위로 장비가 대체로 고정된다. 장별 예외는 [07. 장별 오버라이드](07-per-photo-override.md)로 분리.

## 알려진 한계

- **표시 여부와 입력값이 분리되지 않음** — "렌즈를 기억하는데 이 롤에서는 표시하고 싶지 않다"면 입력값을 지워야 하고, 다시 쓰려면 재입력해야 한다. 항목별 표시 토글 필요.
- **한 줄 레이아웃 고정** — 장비명이 길면 텍스트가 사진 폭을 넘칠 수 있다. 두 줄(윗줄 바디·렌즈 / 아랫줄 필름) 옵션 필요.
- **새로고침 시 입력값 소실** — localStorage 자동 저장 없음.

## 개선 계획 (1단계 — v3 인수인계 명세의 미반영분 구현)

- [ ] 접이식 "표시 설정" 패널: 바디/렌즈/필름 **항목별 표시 체크박스**
- [ ] 한 줄/두 줄 레이아웃 선택 — 두 줄이면 윗줄 바디·렌즈, 아랫줄 필름. 여백 띠 높이 0.078W → 0.112W
- [ ] 촬영 정보 + 표시 설정 localStorage 자동 저장 (store 래퍼 경유, 실패 시 메모리 폴백)
- [ ] 3단계: 날짜/장소 필드 (표시 토글 포함)
