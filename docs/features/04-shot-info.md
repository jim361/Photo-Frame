# 04. 촬영 정보 + 표시 설정

## 목적

카메라/렌즈/필름·메모/날짜를 한 번 입력하면 묶음 전체에 적용된다. 핵심은 **불확실한 정보를 표시하지 않을 수 있는 것** — 필름 사진은 현상까지 시차가 있어 렌즈 기억이 흐려지는 일이 잦고, 틀린 정보를 박제하느니 빼는 쪽을 지원한다 (CONCEPT.md 원칙 2). 디지털 사진은 반대로 EXIF에 확실한 정보가 있으므로 자동으로 읽어온다.

## 현재 동작

- **입력 필드**: 카메라 / 렌즈 / 필름·기타 (텍스트 + `datalist` 자동완성) + **날짜** (자유 형식, 예 `2026.07`) + **설정** (촬영 설정값 — 조리개/셔터/ISO/초점거리, 예 `F1.8 · 1/250s · ISO 400 · 35mm`; 디지털은 EXIF 자동, 필름 유저는 수동 입력) + **메모** (여러 줄 textarea, 최대 4줄 표시). 렌더 시 설정은 필름·기타와 묶여 기술 정보 줄로 나간다. 롤 번호·시작 프레임 필드는 있었으나 사용자 결정으로 제거됨(2026-07).
- **로고·서명**: 이미지 파일(투명 PNG 권장)을 팔레트에 저장(`frame.logos`, 400px 축소 dataURL)해 두고 클릭으로 적용 — 인스타 아이디 서명, 개인 로고 등. 이미지 없이 **텍스트 서명**(예 `@jim361`)을 입력하면 필기체로 같은 자리에 렌더된다(이미지 선택 시 이미지 우선). 위치·크기 %는 표시·레이아웃 설정의 방향별 프로파일에서 조절. 브랜드 로고 번들 대신 사용자 업로드 방식을 택한 것은 상표 이슈 회피 목적.
- **눈 아이콘**: 각 필드 라벨 오른쪽의 눈을 클릭해 항목별 표시/숨김을 바로 토글한다 (숨기면 아이콘에 사선 + 입력 필드가 흐려짐). 이전의 표시 설정 패널 체크박스를 대체.
- **메모 렌더링**: 정보 줄 아래로 줄 단위로 쌓이고, 필요한 만큼 프레임 하단 공간이 자동 확장된다. 필름 스트립·여백 띠는 모노스페이스 저채도, 인스탁스는 손글씨 느낌의 이탤릭 산세리프. 위치는 촬영 정보 위치를 따른다.
- **EXIF 장별 적용**: JPEG를 불러올 때 `readExif()`가 APP1(TIFF IFD)에서 Make/Model/LensModel/DateTimeOriginal을 직접 파싱(의존성 없음, 앞 256KB만 읽음). **LensModel이 있을 때만** 신뢰한다 — 필름 스캐너 기기명(Noritsu 등) 오입력 가드. EXIF는 전역 입력을 건드리지 않고 **그 사진에만 붙는다**(`shot.exif`):
  - 렌더 시 카메라/렌즈/날짜는 그 사진의 값(사용자 수정분 `shot.own` > EXIF)을 쓰고, 전역값으로 폴백하지 않는다 — 필름/디지털 혼합 작업에서 서로 오염되지 않음
  - 입력 필드는 선택된 사진 기준의 뷰: EXIF 사진 선택 중에는 라벨에 **EXIF 배지**가 뜨고, 이때의 편집은 그 사진에만 기록된다. 필름(EXIF 없는) 사진 선택 중의 편집은 전역(`gInfo`)에 기록
  - **배지 클릭 = 그 필드를 EXIF 원본으로 되돌리기** (장별 수정분 삭제)
  - 장별 수정분은 세션 한정(저장 안 함) — 전역 설정만 localStorage에 저장된다
- **자동완성 소스**: 장비 팔레트(`frame.gear`, [05. 프리셋](05-presets.md))에 사용자가 저장한 값.
- **표시·레이아웃 설정** (촬영 정보 그룹 안의 접이식 섹션). **방향별 프로파일**: 아래 항목 중 레이아웃/글자 크기/하단 공간/테두리 여백/위치/가로 글씨는 [세로 사진 | 가로 사진] 탭으로 따로 저장되고, 렌더 시 각 사진이 자기 방향의 프로파일을 자동으로 쓴다 — 가로·세로 혼합 묶음에서 설정이 서로 어긋나지 않음. 섹션을 펼치면 현재 보고 있는 사진의 방향 탭이 기본 선택. 글꼴·비율·내보내기 설정은 묶음 통일을 위해 전역:
  - **한 줄/두 줄 레이아웃** — 두 줄이면 윗줄 장비(바디·렌즈), 아랫줄 필름. 필름 스트립·여백 띠에 적용, 인스탁스는 항상 2단이라 무시
  - **글자 크기 %** (50~200, 기본 100) — 결과물 텍스트 크기 배율
  - **하단 공간 %** (50~250, 기본 100) — 프레임의 텍스트 영역 높이 배율 (필름 아래 바 / 인스탁스 하단 여백·우측 컬럼 / 여백 띠)
  - **테두리 여백 %** (0~300, 기본 100) — 인스탁스의 얇은 흰 테두리 두께 배율. 0이면 사진이 카드 가장자리에 붙는다. 다른 스타일에는 영향 없음
  - **글꼴** — 기본(스타일별 모노/산세리프 조합) / 모노스페이스 / 산세리프 / 세리프 / 필기체. 시스템 폰트 스택만 사용(외부 요청 없음) — 필기체 등은 OS에 따라 다르게 보일 수 있다
  - **가로 인스탁스 글씨** — 가로(항목별 한 줄 스택, 기본) / 세로(회전) — 세로 카드를 실제로 돌린 것처럼 글씨가 오른쪽 여백을 따라 아래→위로 흐른다. 회전 모드는 세로 카드와 동일한 배치(장비/필름 중앙, 날짜 구석, 메모만큼 여백 확장)
  - **촬영 정보 위치 / 날짜 위치** — 각각 독립 지정. 기본값은 스타일별: 필름 스트립 = 정보 오른쪽·날짜 왼쪽, 인스탁스 = 정보 가운데·날짜 우하단, 여백 띠 = 정보 왼쪽·날짜 오른쪽. 가로 인스탁스(우측 컬럼)에서는 항목이 한 줄씩 스택되므로 촬영 정보 위치가 컬럼 내 가로 정렬로 적용되고 날짜 위치는 무시된다
- **개인 설정** (마스트헤드 우측 톱니바퀴 팝오버): 화면 표시만 바꾸는 설정 — 설정 패널 폭(300~600px, CSS 변수)과 UI 배율(90~130%, zoom). `frame.ui`에 별도 저장, 결과물에는 영향 없음.
- **날짜는 독립 요소**: 정보 줄에 붙지 않고 따로 그려진다. 필름 스트립·여백 띠에서는 데이트 스탬프처럼 주황(`--edge`), 인스탁스에서는 카드 톤의 회갈색. 날짜와 정보가 서로 다른 쪽에 있으면 같은 줄의 정보 텍스트가 남는 폭에 맞춰 **자동 축소**된다(최소 55%) — 겹침 방지.
- **빈 칸 = 생략**: 렌더 시 `filter(Boolean)`으로 빈 항목이 텍스트에서 빠진다 (토글과 별개로 동작).
- **즉시 반영**: 모든 입력에 `input` 이벤트 → 미리보기 갱신, 줌 유지.
- **자동 저장**: 촬영 정보(바디/렌즈/필름)·표시 설정·스타일/톤·내보내기 설정을 localStorage `frame.settings`에 저장, 새로고침 시 복원. 배포본의 첫 실행 값은 비어 있고 F5 / 50mm f/1.8G / Portra 400은 입력 예시용 placeholder로만 표시한다.

## 설계 결정

- **자유 텍스트 + datalist** (select 아님) — 장비는 무한히 다양하고, 자동완성은 반복 입력만 줄이면 된다. 프리셋에서 자동완성을 채우는 것으로 "쓸수록 편해지는" 구조.
- **롤 전역 입력이 기본** — 필름은 롤 단위로 장비가 대체로 고정된다. 장별 예외는 [07. 장별 오버라이드](07-per-photo-override.md)로 분리.

## 알려진 한계

- 표시 토글은 롤 전역 — 장별 예외는 [07. 장별 오버라이드](07-per-photo-override.md)에서 해결.
- 한 줄 모드에서 긴 장비명이 프레임 번호와 겹칠 수 있다 (두 줄 전환이 수동).
