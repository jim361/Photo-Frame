# 04. 촬영 정보 + 표시 설정

## 목적

카메라/렌즈/필름·메모/날짜를 한 번 입력하면 묶음 전체에 적용된다. 핵심은 **불확실한 정보를 표시하지 않을 수 있는 것** — 필름 사진은 현상까지 시차가 있어 렌즈 기억이 흐려지는 일이 잦고, 틀린 정보를 박제하느니 빼는 쪽을 지원한다 (CONCEPT.md 원칙 2). 디지털 사진은 반대로 EXIF에 확실한 정보가 있으므로 자동으로 읽어온다.

## 현재 동작

- **입력 필드**: 카메라 / 렌즈 / 필름·메모 (텍스트 + `datalist` 자동완성) + **날짜** (자유 형식 텍스트, 예 `2026.07`). 필름·메모 칸은 필름명이든 ISO·장소·캡션이든 자유롭게 쓰는 칸. 롤 번호·시작 프레임 필드는 있었으나 사용자 결정으로 제거됨(2026-07) — 결과물의 프레임 번호 표기도 함께 제거.
- **EXIF 자동 입력**: JPEG를 불러올 때 `readExif()`가 APP1(TIFF IFD)에서 Make/Model/LensModel/DateTimeOriginal을 직접 파싱(의존성 없음, 앞 256KB만 읽음). **LensModel이 있을 때만** 카메라·렌즈·날짜를 채운다 — 필름 스캐너가 남긴 기기명(Noritsu 등)이 장비 입력을 덮어쓰는 것을 막는 가드. 묶음당 첫 파일 1회, 토스트로 알림.
- **자동완성 소스**: 장비 팔레트(`frame.gear`, [05. 프리셋](05-presets.md)) — 시드 + 사용자가 저장한 값.
- **표시 설정 패널** (접이식 `<details>`):
  - 카메라/렌즈/필름·메모/날짜 **항목별 표시 체크박스** — 값을 지우지 않고 결과물에서만 뺄 수 있다
  - **한 줄/두 줄 레이아웃** — 두 줄이면 윗줄 장비(바디·렌즈), 아랫줄 필름. 필름 스트립·여백 띠에 적용, 인스탁스는 항상 2단이라 무시
  - **글자 크기 %** (50~200, 기본 100) — 결과물 텍스트 크기 배율
  - **하단 공간 %** (50~250, 기본 100) — 프레임의 텍스트 영역 높이 배율 (필름 아래 바 / 인스탁스 하단 여백 / 여백 띠)
  - **촬영 정보 위치 / 날짜 위치** — 각각 독립 지정. 기본값은 스타일별: 필름 스트립 = 정보 오른쪽·날짜 왼쪽, 인스탁스 = 정보 가운데·날짜 우하단, 여백 띠 = 정보 왼쪽·날짜 오른쪽
- **날짜는 독립 요소**: 정보 줄에 붙지 않고 따로 그려진다. 필름 스트립·여백 띠에서는 데이트 스탬프처럼 주황(`--edge`), 인스탁스에서는 카드 톤의 회갈색. 날짜와 정보가 서로 다른 쪽에 있으면 같은 줄의 정보 텍스트가 남는 폭에 맞춰 **자동 축소**된다(최소 55%) — 겹침 방지.
- **빈 칸 = 생략**: 렌더 시 `filter(Boolean)`으로 빈 항목이 텍스트에서 빠진다 (토글과 별개로 동작).
- **즉시 반영**: 모든 입력에 `input` 이벤트 → 미리보기 갱신, 줌 유지.
- **자동 저장**: 촬영 정보(바디/렌즈/필름)·표시 설정·스타일/톤·내보내기 설정을 localStorage `frame.settings`에 저장, 새로고침 시 복원. 첫 실행 기본값은 F5 / 50mm f/1.8G / Portra 400.

## 설계 결정

- **자유 텍스트 + datalist** (select 아님) — 장비는 무한히 다양하고, 자동완성은 반복 입력만 줄이면 된다. 프리셋에서 자동완성을 채우는 것으로 "쓸수록 편해지는" 구조.
- **롤 전역 입력이 기본** — 필름은 롤 단위로 장비가 대체로 고정된다. 장별 예외는 [07. 장별 오버라이드](07-per-photo-override.md)로 분리.

## 알려진 한계

- 표시 토글은 롤 전역 — 장별 예외는 [07. 장별 오버라이드](07-per-photo-override.md)에서 해결.
- 한 줄 모드에서 긴 장비명이 프레임 번호와 겹칠 수 있다 (두 줄 전환이 수동).
