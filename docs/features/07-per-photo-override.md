# 07. 장별 설정 오버라이드 [부분 구현]

> 2026-07: EXIF 기반으로 부분 구현됨 — EXIF가 있는 사진은 카메라/렌즈/날짜를 장별로 갖고(선택 중 편집도 장별, 배지로 원본 복원), 필름 사진은 전역을 쓴다 ([04](04-shot-info.md) 참고). 아래는 나머지 범위(필름·기타/메모/표시 토글의 장별화, EXIF 없는 사진의 장별 지정)의 요구사항.

## 목적

롤 전역 설정을 유지한 채 특정 컷만 다르게 표시한다. 대표 시나리오: 롤 전체는 "F5 / 렌즈 미표시 / Portra 400"인데, 3번 컷만 어떤 렌즈였는지 확실히 기억나서 "50mm f/1.8G 표시"로 덮어쓰고 싶다. 롤 중간에 렌즈를 교체한 경우도 마찬가지다. **이 프로젝트의 핵심 설계 문제** — "불확실한 정보는 표시하지 않는다" 원칙이 장 단위까지 내려온 형태.

## 요구사항

- **전역 기본값 + 장별 오버라이드** 2계층 구조.
  - 각 shot은 `override` 객체(부분 집합)를 가질 수 있고, 렌더 시 `전역 opts ← override` 병합.
  - 오버라이드 가능한 항목: 바디/렌즈/필름 값, 항목별 표시 여부. (스타일·톤·줄 수는 롤 전역 고정 — 한 롤 안에서 프레임 스타일이 섞이는 것은 스코프 밖)
- 전역 값을 바꾸면 오버라이드가 **없는** 장에만 반영된다. 오버라이드된 항목은 전역 변경에 영향받지 않는다.
- 오버라이드 해제(전역으로 되돌리기)가 항목 단위로 가능해야 한다.
- 어떤 장에 오버라이드가 있는지 필름 스트립에서 식별 가능해야 한다 (예: 칩에 점 표시).

## UI 방향 (제안 — 구현 시 재검토)

필름 스트립에서 사진 선택 → 촬영 정보 패널에 **"이 장만 다르게"** 토글이 나타난다. 켜면 현재 장의 값으로 편집 모드가 전환되고, 패널이 시각적으로 구분된다(예: 주황 테두리). 대안으로 검토했던 방식:

- 미리보기 위 인라인 편집 — 발견성은 좋으나 캔버스 위 오버레이 구현 비용이 큼
- 장별 팝오버 — 스트립 칩이 작아 터치 정확도 문제

## 알려진 리스크

- 전역/오버라이드 편집 모드를 사용자가 혼동할 수 있다 — "지금 타이핑하면 어디에 적용되는가"가 항상 명확해야 한다.
- 프리셋 적용 시 오버라이드를 유지할지 초기화할지 결정 필요.

## 개선 계획

- [ ] 3단계: 데이터 구조(override 병합) 구현 — 렌더 엔진 분리(2단계) 후
- [ ] 3단계: UI 확정 및 구현 (구현 전 사용자와 배치 재검토)

## 수용 기준

- [ ] 특정 장만 렌즈 표시를 켜고 전역은 끈 상태를 만들 수 있다
- [ ] 전역 값 변경이 오버라이드된 항목을 건드리지 않는다
- [ ] 항목 단위로 전역 복귀가 가능하다
- [ ] 오버라이드된 장이 스트립에서 구분된다
- [ ] 내보내기 결과에 장별 오버라이드가 반영된다
