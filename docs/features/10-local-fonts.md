# 10. 로컬 글꼴

## 목적

외부 폰트 서버나 업로드 없이 사용자의 PC 안에서만 글꼴을 선택한다. 기본 사용은 권한 요청 없이 시작하고, 필요한 사람만 PC 설치 글꼴 또는 가지고 있는 폰트 파일을 추가한다.

## 현재 동작

- **기본 글꼴**: 기존 시스템 스택을 영문(스타일별/모노/산세리프/세리프/필기체)과 한글(고딕/명조/손글씨)로 나눠 제공한다. 앱이 폰트 파일을 재배포하지 않으며 OS에 따라 실제 글꼴이 달라질 수 있다.
- **PC 글꼴**: 데스크톱 Chromium에서 `PC 글꼴 불러오기`를 누르면 `queryLocalFonts()` 권한을 요청한다. 허용된 글꼴을 family별로 묶고 Regular face를 대표로 표시하며, 선택할 때만 바이트를 메모리에 읽어 고유 별칭의 `FontFace`로 등록한다.
- **파일 글꼴**: TTF·OTF·WOFF·WOFF2를 최대 24MB까지 추가한다. 확장자뿐 아니라 sfnt/WOFF 서명(`00 01 00 00`, `OTTO`, `wOFF`, `wOF2` 등)을 확인하고 SHA-256 기반 ID로 중복을 구분한다.
- 영문·한글 선택은 같은 PC·파일 글꼴 목록을 공유한다. Canvas에서는 혼합 문자열을 문자권별 run으로 나눠 그려 앞쪽 범용 폴백이 한글 선택을 가리지 않게 하며, 선택한 글꼴에 글리프가 없으면 해당 문자권의 시스템 스택으로 폴백한다.
- 글꼴을 불러오는 동안 프로젝트 저장과 내보내기를 잠시 막고, `document.fonts.ready` 뒤 같은 `render()` 경로를 다시 그린다.
- PC·파일 글꼴의 사용 권한은 사용자가 확인해야 한다. 글꼴이나 사진을 외부로 전송하는 코드는 없다.

## 프로젝트 저장

- 기본 글꼴은 설정 ID만 저장한다.
- PC 글꼴은 PostScript 이름과 family가 들어간 선택 ID만 저장하고 글꼴 파일은 복사하지 않는다. 다른 PC에서 찾지 못하면 선택을 보존한 채 기본 글꼴로 임시 표시한다.
- 파일 글꼴은 현재 영문·한글 선택에 실제 사용 중인 것만 dataURL로 포함한다. 같은 파일을 양쪽에 사용하면 한 번만 저장한다.
- 구버전 `version: 1` 프로젝트의 `fonts` 없는 형식을 그대로 연다. 파일 글꼴은 최대 2개·각 24MB·합계 40MB, 프로젝트 전체는 64MB로 제한하며 저장과 열기 양쪽에서 같은 한도를 검사한다.
- 복원에 실패한 글꼴이 있어도 사진 manifest와 다른 설정은 열고 기본 글꼴로 대체한다.

## 알려진 한계

- PC 설치 글꼴 열람은 브라우저 지원과 사용자 권한에 의존한다. 다운로드한 단일 HTML의 `file://` 환경에서는 지원되더라도 보안 정책으로 거절될 수 있으므로 파일 선택을 폴백으로 제공한다.
- 파일 글꼴은 localStorage에 넣지 않는다. 프로젝트로 저장하지 않고 새로고침하면 다시 추가해야 한다.
- PC 글꼴은 같은 이름이어도 PC마다 버전이 달라 결과가 완전히 같다고 보장할 수 없다.
- 한글 전체 글리프 포함 폰트는 프로젝트 JSON 용량을 크게 늘릴 수 있다(Base64 약 33% 증가).

## 수용 기준

- [x] 첫 실행은 글꼴·폴더 권한 없이 시작한다
- [x] 미지원 브라우저에서도 파일 글꼴을 추가할 수 있다
- [x] 선택한 파일 글꼴만 프로젝트에 포함되고 사진 원본은 포함되지 않는다
- [x] PC 글꼴을 찾지 못해도 선택 ID가 사라지지 않고 기본 글꼴로 렌더된다
- [x] 미리보기와 내보내기가 같은 등록 글꼴과 `render()` 경로를 쓴다
