# FilmFrame — 컨셉

## 문제 정의

필름으로 사진을 찍고 인스타그램에 올리는 흐름에는 두 가지 마찰이 있다.

1. **매 장 반복되는 수작업.** 필름 스캔본에 프레임(필름 스트립, 인스탁스 카드, 여백 띠)과 촬영 정보를 얹으려면 라이트룸 프리셋이나 포토샵 액션으로도 장마다 텍스트를 고쳐야 한다. 롤 하나(5~36장)를 올리는 데 이 작업이 병목이 된다.
2. **기억나지 않는 촬영 정보.** 필름은 촬영과 결과 확인 사이에 시차가 있다. 현상소에서 스캔본이 돌아올 때쯤이면 어떤 컷을 어떤 렌즈로 찍었는지 기억나지 않는 경우가 많다. 그런데 기존 도구들은 "정보를 채워 넣는 것"만 지원하지, "확실하지 않은 정보를 빼는 것"을 지원하지 않는다.

FilmFrame은 이 두 마찰을 없애기 위해 만든 **로컬 웹 도구**다. 제작자 본인이 Nikon F5로 필름 풍경 사진을 찍고 롤 단위 회고 형식으로 인스타그램에 올리면서 겪은 문제에서 출발했다.

## 대상 사용자

- 필름 사진을 인스타그램에 올리는 사람
- 롤 하나를 한 번에 처리하고 싶은 사람 (장 단위가 아니라 **롤 단위** 작업 흐름)
- 사진을 외부 서버에 올리고 싶지 않은 사람

## 핵심 원칙

1. **사진은 브라우저 밖으로 나가지 않는다.** 서버 없음, 계정 없음, 업로드 없음. 모든 합성은 브라우저의 Canvas에서 일어나고, 결과물은 사용자의 디스크로만 나간다. analytics를 포함해 사진 데이터를 외부로 보내는 코드는 어떤 이유로도 넣지 않는다.
2. **불확실한 정보는 표시하지 않을 수 있다.** 렌즈가 기억나지 않으면 렌즈만 빼고 바디와 필름만 표시한다. 항목별 표시 제어가 이 도구가 기존 대안과 갈라지는 지점이다.
3. **롤 단위 일괄 처리가 기본 흐름이다.** 여러 장을 한 번에 불러오고, 프레임 번호는 순서대로 자동 증가하고, 전체를 한 번에 내보낸다. 장 단위 작업은 예외 경로다.

## 기존 대안과의 비교

| 대안 | 한계 |
|---|---|
| 라이트룸 프리셋 / 포토샵 액션 | 프레임은 얹어주지만 텍스트는 장마다 수동 수정. 롤 단위 흐름 없음 |
| 유료 프레임 앱 (모바일) | 사진을 서버에 업로드하는 경우가 많음. 장비 정보 세밀 제어 어려움 |
| 직접 템플릿 제작 | 초기 1회 비용은 낮지만 매 롤 반복 비용이 그대로 남음 |

## 스코프에서 제외하는 것

**이 도구는 프레임만 얹는다.** 색 보정, 크롭, 리터칭, 노출 조정 등 편집 기능은 만들지 않는다. 편집은 스캔 단계나 전용 도구의 몫이고, FilmFrame은 "완성된 스캔본 → 업로드용 이미지"의 마지막 한 단계만 담당한다. 기능 요청이 편집 쪽으로 기울면 이 문서를 근거로 거절한다.
