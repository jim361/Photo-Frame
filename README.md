# PhotoFrame

사진에 프레임과 촬영 정보를 얹어 인스타그램 업로드용 이미지를 만드는 **로컬 웹 도구**.
필름 스캔이든 디지털 사진이든 — 서버 없음, 계정 없음, **사진은 브라우저 밖으로 나가지 않습니다.**

**▶ 바로 쓰기: https://jim361.github.io/Photo-Frame/**

## 왜 만들었나

Nikon F5로 필름 풍경 사진을 찍고 롤 단위로 인스타그램에 올리다가 만들었습니다. 현상소에서 스캔본이 돌아올 때쯤이면 **어떤 컷을 어떤 렌즈로 찍었는지 기억나지 않는** 경우가 많았습니다. 기존 도구(라이트룸 프리셋, 포토샵 액션, 유료 앱)는 매 장 텍스트를 고쳐야 하거나 사진을 서버에 올려야 했고, 무엇보다 "확실하지 않은 정보를 빼는 것"을 지원하지 않았습니다.

필름에서 출발했지만 지금은 **모든 사진**이 대상입니다 — 디지털 사진은 EXIF에서 카메라·렌즈·촬영일을 자동으로 읽어옵니다.

- **불확실한 정보는 표시하지 않을 수 있습니다.** 렌즈가 기억나지 않으면 체크 하나로 렌즈만 뺍니다.
- **묶음 단위가 기본 흐름입니다.** 한 번에 불러오고, 드래그로 순서를 정리하고, 한 번에 내보냅니다.
- **사진이 어디로도 전송되지 않습니다.** 합성은 전부 브라우저 Canvas에서 일어납니다.

자세한 배경은 [docs/CONCEPT.md](docs/CONCEPT.md).

## 사용법

1. **열기** — 배포 페이지를 열거나, `index.html`을 내려받아 브라우저로 엽니다. 설치·빌드 불필요.
2. **사진 놓기** — JPG/PNG를 드래그하거나 클릭해서 여러 장 선택. 디지털 사진이면 EXIF에서 카메라·렌즈·촬영일이 자동 입력됩니다.
3. **스타일 선택** — 필름 스트립 / 인스탁스 / 하단·상단 여백.
4. **촬영 정보 입력** — 카메라/렌즈/필름·기타/날짜, 그리고 사진 아래 들어갈 여러 줄 **메모**. 각 항목 옆 **눈 아이콘**으로 표시/숨김을 바로 토글. 우상단 **톱니바퀴(고급 설정)**에서 한 줄/두 줄 레이아웃, 글자 크기, 하단 공간, 테두리 여백(인스탁스), 글꼴, 가로 인스탁스 글씨 방향(가로 스택/세로 회전), 정보·날짜 위치를 각각 조절. 자주 쓰는 값은 프리셋 그리드에 ＋로 저장해 두고 클릭으로 적용.
5. **내보내기** — 저장 폴더를 지정(Chrome/Edge)하고 비율 맞춤·크기·형식(PNG/JPEG)을 골라 전체 내보내기. 기본값 "인스타 자동"은 피드 허용 비율(4:5~1.91:1)을 벗어나는 결과물에만 여백을 붙여 업로드 시 잘림을 막습니다. 파일명은 `원본이름_framed.jpg`로 원본과 나란히 정렬됩니다.

단축키: `+`/`−` 줌, `0` 화면 맞춤, `1` 원본 크기.

## 프라이버시 원칙

- 사진 데이터를 외부로 보내는 코드는 **어떤 이유로도 넣지 않습니다** (analytics 포함).
- 저장되는 것은 장비 프리셋과 입력 설정(localStorage)뿐이며, 이것도 사용자의 브라우저 안에만 있습니다.

## 브라우저 지원

| 기능 | Chrome / Edge | Firefox | Safari |
|---|---|---|---|
| 프레임 합성·내보내기 | ✅ | ✅ | ✅ |
| 폴더 지정 저장 | ✅ | ❌ 다운로드 폴백 | ❌ 다운로드 폴백 |
| EXIF 자동 회전 | ✅ | ✅ | ✅ |
| EXIF 촬영 정보 자동 입력 | ✅ JPEG | ✅ JPEG | ✅ JPEG |
| 터치 드래그 정렬 / 핀치 줌 | 미구현 | 미구현 | 미구현 |

- 폴더 지정 저장(File System Access API)은 Chromium 전용입니다. 다른 브라우저는 장당 다운로드로 폴백됩니다.
- 색 프로파일: Canvas가 임베디드 ICC 프로파일을 무시하므로 **sRGB로 스캔/변환된 파일**을 권장합니다.

## 기술적 결정

- **단일 HTML, 의존성 0** — 내려받아 더블클릭으로 여는 사용 방식을 지키기 위해. 이후 구조 전환(Vite+TS)에서도 빌드 결과는 단일 HTML을 유지할 계획입니다.
- **미리보기 = 내보내기** — 같은 `render()` 함수를 쓰므로 보이는 그대로 저장됩니다.
- **원본 해상도로 합성 후 축소** — 텍스트·퍼포레이션의 선명도를 지키는 순서. 브라우저 캔버스 한계를 넘는 초대형 스캔은 자동 축소 후 안내합니다.
- 구조와 마이그레이션 계획: [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) · 디자인 시스템: [docs/DESIGN.md](docs/DESIGN.md) · 기능별 명세: [docs/features/](docs/features/)
